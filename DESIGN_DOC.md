## 一、项目概述

**AI视觉精灵** 是一款基于 Web 的 AI 视觉对话应用，允许 AI 通过摄像头"看到"用户所处的环境，同时通过麦克风"听到"用户的语音输入，实现多模态自然交互。

- **视觉理解**：Qwen-VL-Max（通义千问视觉模型）
- **对话生成**：DeepSeek-Chat
- **语音识别**：浏览器内置 Web Speech API（零成本）
- **语音合成**：浏览器内置 Speech Synthesis API（零成本）
- **架构**：Python Flask + 原生 HTML/CSS/JS
- **零外部依赖**：图标（内嵌SVG精灵表）、字体（系统字体栈），无任何CDN请求

### 2.2 核心功能详解

**US-01～US-02 摄像头与视觉分析**
- 使用 `navigator.mediaDevices.getUserMedia()` 获取摄像头流
- 通过 Canvas 将视频帧截取为 Base64 JPEG
- 发送至 `/api/vision`，调用 Qwen-VL-Max 进行视觉理解
- 分析结果显示在左侧"画面分析结果"面板中

**US-03～US-04 多模态对话**
- 文字输入通过 `/api/analyze_and_chat` 接口，同时发送画面截图和用户消息
- 语音输入使用浏览器内置 `SpeechRecognition` API，无需付费 ASR 服务
- 后端先调用 Qwen-VL 理解画面，再调用 DeepSeek-Chat 结合画面+消息生成回复

**US-05 语音合成（TTS）**
- 使用浏览器内置 `SpeechSynthesis` API
- AI回复通过打字机效果逐字显示，同时自动分句朗读
- 支持3种音色切换：通用款（柔和）、成熟款（沉稳）、温暖款（温柔，默认）
- 发送新消息或开启麦克风时自动停止当前朗读
- 语音指令"别读了"停止朗读，"继续读"恢复朗读
- 点击音量按钮可手动切换朗读/静音
- 零 API 调用成本

**US-05扩展：TTS竞态控制**
- `stopTts()` 取消当前朗读并清空队列
- 打字机通过 `typewriterId` 机制管理：新消息自动取消旧打字机，防止旧内容混入TTS队列
- `SpeechSynthesisUtterance.onend` 回调增加守卫判断，防止被取消的句子触发下一句朗读

**US-11 打字机效果**
- AI回复逐字显示，随机微调速度模拟真人打字
- 遇到标点符号自动停顿，增强阅读节奏感
- 打字机与TTS联动：每打完一句自动送入朗读队列

**US-12 AI连接状态**
- 顶部导航栏AI引擎按钮：绿色=已连接，红色=未连接，黄色=检测中
- 页面加载时自动检测 `/api/config` 接口
- 点击按钮可重新检测连接状态
- 与摄像头按钮同款渐变样式，视觉统一

**US-24 对话风格**
- 支持4种风格切换：默认（温暖自然）、可爱（活泼）、专业（严谨）、简洁（高效）
- 风格通过 system prompt 注入，影响AI回复的语气和措辞

**US-25 联网搜索**
- 点击地球图标开启联网搜索
- 使用 DuckDuckGo 免费搜索，无需API密钥
- 搜索结果作为上下文注入对话，AI可引用实时信息
- 内置超时（8秒）和重试（1次）机制，搜索结果去重截断

**US-26 保存截图与对话**
- 将当前摄像头画面保存为JPEG文件
- 同步保存完整对话记录为TXT文件
- 文件存储在 `saved_screenshots/` 目录，按时间戳命名

**US-06 自动分析**
- 默认每 3 秒自动分析一次（可调 2/3/5/10 秒）
- 独立于对话，持续更新画面分析结果
- 用户可随时关闭

**US-09 多轮对话**
- 服务端维护会话级对话历史（`CONVERSATION_HISTORY` 字典）
- 每轮保留最多 20 条历史消息
- 视觉上下文作为 system message 注入

## 三、运营成本控制策略

### 3.1 策略全景

| 编号 | 策略 | 是否采用 | 节省目标 |
|:----:|------|:--------:|----------|
| C-01 | **前端 JPEG 压缩**：截图 Quality 60%，减少传输和 API token 消耗 | ✅ | Token 节省 ~60% |
| C-02 | **分辨率限制**：分析用图片限制 640×480 | ✅ | Token 进一步节省 |
| C-03 | **自动分析间隔控制**：默认 3 秒，可调至 5/10 秒或关闭 | ✅ | API 调用频率可控 |
| C-04 | **对话历史截断**：保留最近 20 轮，超出自动丢弃 | ✅ | 防止 Token 无限增长 |
| C-05 | **视觉分析缓存**：同一画面不重复分析 | ❌ | 画面实时变化，命中率低 |
| C-06 | **按需分析模式**：默认手动触发，非自动分析 | ✅ | 避免无意义调用 |
| C-07 | **回复长度限制**：System prompt 要求 200 字以内 | ✅ | 每次回复 Token 可控 |
| C-08 | **Token 上限限制**：视觉 500-800，对话 600 | ✅ | 单次调用成本有上限 |
| C-09 | **模型分级选型**：视觉用 Qwen，对话用 DeepSeek | ✅ | 各取所长，成本最优 |
| C-10 | **浏览器内置 ASR**：Web Speech API，零成本 | ✅ | 省去付费 ASR 服务 |
| C-11 | **浏览器内置 TTS**：Speech Synthesis API，零成本 | ✅ | 省去付费 TTS 服务 |
| C-12 | **流式输出（Streaming）** | ✅️ | 打字机效果逐字显示，与TTS联动分句朗读 |
| C-13 | **请求合并**：`/api/analyze_and_chat` 一次完成视觉+对话 | ✅ | 减少 HTTP 往返 |
| C-14 | **本地模型降级** | ❌ | 运维复杂度高 |
| C-15 | **用量监控与告警** | ❌ | 后续版本添加 |

> **采用率：12/15 = 80%**

### 3.2 成本估算

以典型使用场景（每天 1 小时，自动分析 3 秒/次，对话 30 次/小时）估算：

| 成本项 | 单价（估算） | 用量 | 日成本 |
|--------|:-----------:|------|:------:|
| Qwen-VL-Max 视觉分析 | ~¥0.02/次 | 1,200 次 | ~¥24 |
| DeepSeek-Chat 对话 | ~¥0.001/次 | 30 次 | ~¥0.03 |
| ASR 语音识别 | ¥0（浏览器内置） | 30 次 | ¥0 |
| TTS 语音合成 | ¥0（浏览器内置） | 30 次 | ¥0 |
| **合计** | | | **~¥24/天** |

> 实际使用中，自动分析通常不会持续开启 1 小时。手动模式下日成本可控制在 ¥1 以内。

### 3.3 核心省钱技巧总结

1. **语音全免费**：ASR + TTS 全部走浏览器内置能力，零额外开销
2. **图片极限压缩**：JPEG Quality 60 + 640×480，单张图片从 ~2MB 压到 ~30KB
3. **模型选型精准**：Qwen-VL 做视觉（多模态能力强），DeepSeek 做对话（极致性价比，约 ¥1/百万 token）
4. **按需调用**：默认不自动分析，所有 API 调用都由用户主动触发或明确开启
5. **Token 严格管控**：每层都有上限（视觉 800、对话 600、历史 20 轮、回复 200 字）

### 4.2 API 端点

| 端点 | 方法 | 功能 | 调用模型 |
|------|:----:|------|----------|
| `/api/vision` | POST | 纯视觉分析，返回画面描述 | Qwen-VL-Max |
| `/api/chat` | POST | 纯文本对话（可附带视觉上下文，支持联网搜索） | DeepSeek-Chat |
| `/api/analyze_and_chat` | POST | 视觉分析 + 对话一次完成 | Qwen-VL + DeepSeek |
| `/api/search` | POST | 联网搜索，返回搜索结果摘要 | DuckDuckGo（免费） |
| `/api/save_screenshot` | POST | 保存当前截图和对话记录到本地 | — |
| `/api/reset` | POST | 重置指定会话的对话历史 | — |
| `/api/config` | GET | 查询 API 配置状态，供前端检测 AI 连接 | — |

### 4.3 数据流

```
用户点击发送 / 语音输入结束
  │
  ├─→ 前端: Canvas 截取当前帧 → toDataURL('image/jpeg', 0.6)
  │
  ├─→ POST /api/analyze_and_chat
  │     body: { image: "data:image/jpeg;base64,...", message: "用户说的话", session_id: "xxx" }
  │
  ├─→ 后端:
  │     ① 解析 Base64 图片
  │     ② 调用 Qwen-VL-Max (MultiModalConversation)
  │        → 返回视觉描述文本
  │     ③ 构建 messages: [system_prompt, vision_context, ...history, user_message]
  │     ④ 调用 DeepSeek-Chat (chat.completions.create)
  │        → 返回对话回复
  │     ⑤ 更新对话历史
  │
  └─→ 返回前端: { success: true, reply: "...", vision_context: "..." }
       │
       ├─→ 打字机逐字显示 AI 回复（typewriterMessage）
       ├─→ 每遇到句末标点（。！？等）自动分句
       ├─→ speakStreamingText 将句子送入 TTS 队列
       ├─→ speakNextInQueue 逐句朗读（SpeechSynthesis）
       ├─→ 更新画面分析面板
       │
       └─→ 用户交互控制:
             ├─ 点击麦克风/发送新消息 → stopTts() + typewriterId++（取消旧朗读）
             ├─ 语音指令"别读了" → ttsPausedByUser = true
             └─ 语音指令"继续读" → ttsPausedByUser = false
```

### 4.4 联网搜索流程

```
用户开启联网搜索 + 发送消息
  │
  ├─→ POST /api/chat  { enable_search: true, message: "..." }
  │
  ├─→ 后端:
  │     ① web_search(message)  →  DuckDuckGo
  │        ├─ ThreadPoolExecutor 超时控制 (8s)
  │        ├─ 失败重试 1 次
  │        └─ 结果去重 + 截断 (每条 ≤200 字)
  │     ② 搜索结果注入 system prompt
  │     ③ 调用 DeepSeek-Chat 生成回复
  │
  └─→ 返回: { reply: "...", search_used: true, search_context: "..." }
       │
       └─→ 前端展示可折叠的搜索来源卡片
```

## 五、关键设计决策

| 决策点 | 方案 A | 方案 B | 选择 | 理由 |
|--------|--------|--------|:----:|------|
| 视觉模型 | GPT-4V | Qwen-VL-Max | **Qwen** | 国内访问稳定，多模态能力强 |

| 语音合成 | 微软 Azure TTS | Speech Synthesis | **浏览器内置** | 零成本，浏览器原生 |
| 图标方案 | Font Awesome | 内嵌 SVG 精灵 | **内嵌 SVG** | 零外部依赖，离线可用 |
| 字体方案 | Google Fonts | 系统字体栈 | **系统字体栈** | 零外部依赖，加载即时 |
| 前端框架 | React/Vue | 原生 HTML/JS | **原生** | 无构建步骤，快速部署 |
| 视频流 | WebRTC | getUserMedia | **getUserMedia** | 无需信令服务器 |
